## Problem Statement

Every student has the same exam-week problem: a folder of PDFs and no way to test whether they actually understand them. StudyBuddy Pro turns passive notes into active recall, and tells the student exactly what they still don't know.

## What It Does

1. **Upload & Index** — Accepts multiple PDF study documents, splits them into chunks, and stores them in a searchable vector database.
2. **Ask Questions** — Answers free-text questions about your notes with document + page citations, so every answer is traceable back to the source.
3. **Generate a Quiz** — Creates a 10-question quiz (5 MCQ + 5 short-answer) sourced entirely from your uploaded content, returned as structured JSON.
4. **Get Revision Summaries** — After grading, identifies which topics you missed and writes a focused 2-paragraph revision summary for each, using only your own material.
5. **Track Progress (Stretch Goal)** — Logs every quiz attempt in the session and highlights topics you keep missing across multiple tries.

## Approach / Methodology

- PDFs are loaded and split using a `RecursiveCharacterTextSplitter` (chunk size 500, overlap 100) to preserve context across chunk boundaries.
- Each chunk is embedded using the `all-MiniLM-L6-v2` sentence-transformer model and stored in a **ChromaDB** vector store, with the source filename and page number kept as metadata.
- Questions are answered via **retrieval-augmented generation (RAG)**: the top matching chunks are retrieved and passed to a Groq-hosted LLaMA 3.3 70B model, which is instructed to answer *only* from that context and cite its source.
- Quizzes are generated by retrieving a broad sample of the document, then prompting the LLM to return strictly structured JSON (parsed and validated in code, not just displayed as text).
- Revision summaries re-run retrieval specifically on each weak topic, so the explanation is grounded in the same source material rather than general knowledge.

## Dataset / Data Sources

No external dataset is used — the app is designed to work with **any PDF the user uploads** (lecture notes, textbook chapters, etc.). All answers, quizzes, and summaries are grounded exclusively in the user's own uploaded documents.

## Setup Instructions (Run Locally)

1. Clone this repository:
```bash
   git clone https://github.com/Khushi2007Shah/studybuddy-pro-.git
   cd studybuddy-pro-
```
2. Create and activate a virtual environment:
```bash
   python -m venv venv
   source venv/bin/activate   # Windows: venv\Scripts\activate
```
3. Install dependencies:
```bash
   pip install -r requirements.txt
```
4. Create a `.env` file in the project root with your Groq API key:
GROQ_API_KEY=your_key_here
5. Run the app:
```bash
   python app.py
```
6. Open the local URL shown in the terminal (usually `http://127.0.0.1:7860`).

## Known Limitations

- Short-answer quiz grading uses loose substring matching rather than semantic similarity, so slightly different phrasing may occasionally be marked incorrect.
- Progress tracking only persists for the current session (in-memory), and resets if the app restarts.
- Very large PDFs may take longer to index on first upload due to embedding generation time.
